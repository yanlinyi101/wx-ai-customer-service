## 一、打包代码

在本机打开终端，进入项目目录，安装依赖到项目内（云函数需要依赖随代码上传）：

```bash
cd C:\Users\EDY\Desktop\wechat_ai_service

# 把依赖安装到项目目录下的 site-packages 文件夹
pip install -r requirements.txt -t ./site-packages
```

然后选中以下所有文件，压缩为 `wechat_ai_service.zip`：

```
main.py
config.py
crypto.py
wechat_api.py
ai_service.py
scf_bootstrap
requirements.txt
site-packages/（整个文件夹）
```

> Windows 用户：全选这些文件 → 右键 → 压缩为 ZIP 文件

## 二、创建云函数

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com)
2. 搜索并进入 **云函数 SCF**
3. 点击 **新建**，填写：

| 字段 | 填写内容 |
|---|---|
| 函数类型 | **Web 函数** |
| 函数名称 | `wechat-ai-service`（自定义） |
| 地域 | 上海 或 广州 |
| 运行环境 | **Python 3.11** |
| 提交方式 | **本地上传 ZIP 包** |
| 启动文件 | `scf_bootstrap` |

4. 上传第一步打包好的 `wechat_ai_service.zip`
5. **执行超时时间**改为 `30` 秒（默认3秒不够 AI 响应）
6. 内存配置保持 **128MB** 即可

---

## 三、配置环境变量

在云函数配置页面，找到 **环境变量** 一栏，逐个添加（不要使用 .env 文件）：

| 变量名 | 值 | 来源 |
|---|---|---|
| `WECHAT_TOKEN` | 你自己设定的字符串（如 `mytoken123`） | 自定义，稍后填入微信后台 |
| `WECHAT_APP_ID` | 小程序 AppID | mp.weixin.qq.com → 开发 → 开发设置 |
| `WECHAT_APP_SECRET` | 小程序 AppSecret | 同上（需要验证身份） |
| `WECHAT_ENCODING_AES_KEY` | 43位随机字符串 | 在微信后台点"随机生成"获得 |
| `AI_API_KEY` | DeepSeek API Key | platform.deepseek.com |
| `AI_BASE_URL` | `https://api.deepseek.com` | 固定值 |
| `AI_MODEL` | `deepseek-chat` | 固定值 |
| `KF_ACCOUNT` | 指定客服账号（可选，留空则自动分配） | 格式见下方说明 |

> **`KF_ACCOUNT` 格式说明：**
> - 留空（默认）：触发转人工后由微信平台自动分配空闲客服
> - 填写指定账号：格式为 `客服微信号@公众号ID`，例如 `kefu01@gh_xxxxxxxx`
> - 公众号ID（即 `gh_` 开头的字符串）可在 mp.weixin.qq.com → 设置 → 公众号设置 → 账号信息 中查看

## 四、开启函数URL并获取 Webhook 地址

> ⚠️ **注意**：API 网关触发器已于2024年7月停止新建，**不要进入"触发管理"**，直接按以下步骤开启"函数URL"。

函数创建成功后，进入函数详情页，找到顶部标签栏中的 **「函数URL」** 标签（与"函数管理"、"触发管理"并列），点击进入：

1. 点击 **「开启」** 按钮
2. 鉴权方式选择 **「无鉴权」**（微信自带签名验证，无需额外鉴权）
3. 点击 **「确认」** 保存

页面随即显示一个 HTTPS 地址，格式如：
```
https://xxxxxxxx.scf.tencentcs.com/
```

你的 **Webhook URL** 就是在这个地址末尾加上 `webhook`：
```
https://xxxxxxxx.scf.tencentcs.com/webhook
```

> 自带 HTTPS，无需购买域名或申请证书。
> 目前为测试域名可直接使用，后期正式使用如需绑定自定义域名需要 ICP 备案。

**找不到"函数URL"标签？** 可能是旧版控制台，改用以下路径：函数详情 → **「函数配置」** → 滚动到底部 → **「函数URL」** → 点击编辑开启。

## 五、配置微信小程序后台

登录 [mp.weixin.qq.com](https://mp.weixin.qq.com)，进入：**开发 → 开发管理 → 消息推送**

按以下顺序操作（顺序很重要）：

---

### 第1步：填写 URL

```
https://xxxxxxxx.scf.tencentcs.com/webhook
```
填入第四步从 SCF 控制台获得的函数URL地址，末尾必须是 `/webhook`。

---

### 第2步：填写 Token（令牌）

自己随便设一个字符串，规则：
- 只能包含英文字母和数字
- 长度 3-32 位
- 示例：`YidorService2026`

**⚠️ 记下这个值**，后面要填入 SCF 环境变量 `WECHAT_TOKEN`。

---

### 第3步：生成 EncodingAESKey

点击 **随机生成** 按钮，页面会自动填入一个 43 位的随机字符串。

**⚠️ 立刻复制这个 43 位字符串**，去 SCF 控制台把环境变量 `WECHAT_ENCODING_AES_KEY` 更新为这个值，保存后再回来继续。

> 必须先更新 SCF 环境变量，否则微信验证时服务器无法正确解密，验证会失败。

---

### 第4步：选择消息加密方式

选择 **安全模式（推荐）**

> ⚠️ 不要选明文模式或兼容模式，代码仅支持安全模式。

---

### 第5步：选择数据格式

选择 **XML**

> ⚠️ 必须选 XML，不能选 JSON。代码使用 XML 格式解析消息，选 JSON 会导致解析失败。

---

### 第6步：提交

确认以上全部填写正确后，点击 **提交**。

微信会立即向你的 URL 发一个 GET 验证请求。SCF 服务器响应正确后，显示 **"提交成功"** ✅

如果提交失败，检查：
- SCF 函数是否已部署并正常运行（先访问 `/health` 确认）
- Token 是否与 SCF 环境变量 `WECHAT_TOKEN` 完全一致（区分大小写）
- URL 末尾是否有 `/webhook`

## 八、配置转人工客服

用户发送"转人工"、"人工客服"等关键词后，系统会：
1. 回复一条提示文字
2. 调用微信 API 将对话转接给人工坐席
3. 此后该用户的消息**不再触发 Webhook**，直接进入微信客服平台

### 第0步（关键）：开启官方客服工作台收发消息

登录 [mp.weixin.qq.com](https://mp.weixin.qq.com)，进入：**功能 → 客服**

找到页面顶部的开关：

> **"在官方客服工作台收发消息"** → 打开 ✅

> ⚠️ **此开关默认关闭。** 关闭时，微信会将所有消息仅转发到第三方服务器（Webhook），完全绕过客服工作台路由，导致 `transfer_customer_service` 被动回复无效，用户永远不会出现在"待接入用户"列表中。
>
> 必须先打开此开关，转人工功能才能生效。

---

### 第1步：在微信后台添加客服账号

登录 [mp.weixin.qq.com](https://mp.weixin.qq.com)，进入：**客服 → 客服成员 → 添加客服成员**

- 填写客服昵称
- 绑定客服人员的微信号（对方需接受邀请）
- 添加后，该微信号会在"微信客服"App 或微信内收到接入通知

> ⚠️ **此步骤为必须操作**，否则转接 API 调用成功，但无人接单，用户会陷入无响应状态。

### 第2步：选择分配方式

**方式A：自动分配（推荐新手）**

将环境变量 `KF_ACCOUNT` 留空（不填或删除该变量），微信平台会自动分配当前空闲的客服。

**方式B：指定客服账号**

在 SCF 环境变量中添加：

| 变量名 | 值 |
|---|---|
| `KF_ACCOUNT` | `客服微信号@公众号ID` |

其中公众号ID（`gh_` 开头）在 mp.weixin.qq.com → **设置 → 账号信息** 中查看。

示例：`KF_ACCOUNT` = `kefu01@gh_abc123def456`

### 第3步：验证转接效果

1. 部署新版代码后，在小程序客服对话中发送 **"转人工"**
2. 应先收到提示文字
3. 绑定的客服微信收到接入通知
4. 此后再发消息**不会触发 AI 回复**，消息直接进入人工坐席
5. 云函数日志应出现 `[转人工] 转接成功`

## 九、聊天记录永久存储（COS）

> 默认关闭（`COS_ENABLED=false`），开启后每条对话自动写入腾讯云 COS，按天归档，可随时下载查看。

### 第1步：创建 COS 存储桶

1. 登录 [腾讯云 COS 控制台](https://console.cloud.tencent.com/cos)
2. 点击 **创建存储桶**，填写：

| 字段 | 填写内容 |
|---|---|
| 名称 | `wechat-chat-logs`（自定义） |
| 所属地域 | 与云函数 **同地域**（如：广州） |
| 访问权限 | **私有读写**（推荐） |

3. 创建完成后，记下 **存储桶名称**，格式为 `bucketname-appid`（例：`wechat-chat-logs-1251234567`）

### 第2步：创建子账号并获取密钥

1. 进入 [访问管理 CAM](https://console.cloud.tencent.com/cam)
2. **用户 → 新建用户 → 自定义创建 → 可编程访问**
3. 赋予权限：搜索 `QcloudCOSDataWrite`，勾选后保存
4. 创建完成后，下载或复制 **SecretId** 和 **SecretKey**（只显示一次）

> ⚠️ 使用子账号而非主账号密钥，权限最小化，更安全。

### 第3步：在 SCF 环境变量中添加以下配置

| 变量名 | 值 | 说明 |
|---|---|---|
| `COS_ENABLED` | `true` | 开启日志写入（填 `false` 则禁用） |
| `COS_SECRET_ID` | 第2步获取的 SecretId | 子账号密钥 |
| `COS_SECRET_KEY` | 第2步获取的 SecretKey | 子账号密钥 |
| `COS_REGION` | `ap-guangzhou`（与存储桶同地域） | 地域标识 |
| `COS_BUCKET` | `wechat-chat-logs-1251234567` | 存储桶完整名称（含 AppID） |

### 第4步：重新打包并部署代码

修改了 `requirements.txt`（新增 `cos-python-sdk-v5`），需要重新安装依赖并打包：

```bash
pip install -r requirements.txt -t ./site-packages
```

将 `cos_logger.py` 也加入打包文件列表：

```
main.py
config.py
crypto.py
wechat_api.py
ai_service.py
cos_logger.py       ← 新增
scf_bootstrap
requirements.txt
site-packages/
```

### 第5步：查看聊天日志

部署完成后发送一条消息，打开 COS 控制台：

**存储桶 → 文件列表 → `chat_logs/` → 按日期文件夹 → 下载 JSON 文件**

每个 JSON 文件内容示例：
```json
{
  "time":   "2026-02-28 14:30:05",
  "openid": "abc12345...",
  "user":   "你们的退款政策是什么？",
  "ai":     "您好，我们支持7天无理由退款..."
}
```

云函数日志中应出现：`[COS日志] 写入成功 chat_logs/2026-02-28/14-30-05_abc12345.json`

## 常见问题

**Q：微信验证服务器失败？**
- 检查 Webhook URL 是否包含 `/webhook` 路径
- 确认环境变量 `WECHAT_TOKEN` 与微信后台填写的完全一致
- 在云函数日志中查看是否有收到 GET 请求

**Q：收到消息但没有 AI 回复？**
- 检查环境变量 `AI_API_KEY` 是否正确（注意前后无空格）
- 查看云函数日志中是否有 `[AI] 调用失败` 的报错
- 确认 DeepSeek 账户余额充足

**Q：函数超时？**
- 确认执行超时时间已设置为 **30秒**
- DeepSeek 正常响应在5秒内，如持续超时可检查网络连通性

**Q：冷启动慢，用户等待太久？**
- 可在云函数配置中开启**预置并发**（0元版每月有一定免费额度）
- 或忽略此问题：冷启动约1-2秒，加上AI响应仍在可接受范围内
